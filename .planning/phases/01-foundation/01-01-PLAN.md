  <action>
1. **Download Switzer Variable Font:**
   Download the Switzer variable font file from Fontshare. Create directory `app/fonts/` and place `Switzer-Variable.woff2` there. The file should be approximately 70-90KB covering weight range 100-900.

   Use curl or a download command:
   ```
   mkdir -p app/fonts
   curl -L "https://api.fontshare.com/v2/css?f[]=switzer@1&display=swap" to inspect the URL, then download the .woff2 file directly.
   ```
   If the direct download URL is not easily obtainable via CLI, create a placeholder approach: download from the Fontshare API. The URL pattern for Fontshare variable font files is typically at `https://api.fontshare.com/v2/css?f[]=switzer@1`. Parse the CSS response to find the woff2 URL and download it.

   Alternative: If automated download fails, create a checkpoint noting the font file must be manually downloaded from https://www.fontshare.com/fonts/switzer and placed at `app/fonts/Switzer-Variable.woff2`.

2. **Create `app/fonts.ts`:**
   ```ts
   import localFont from 'next/font/local'
   import { Public_Sans } from 'next/font/google'

   export const switzer = localFont({
     src: './fonts/Switzer-Variable.woff2',
     variable: '--font-switzer',
     display: 'swap',
     weight: '100 900',
   })

   export const publicSans = Public_Sans({
     subsets: ['latin'],
     variable: '--font-public-sans',
     display: 'swap',
   })
   ```

3. **Update `tailwind.config.ts`:**
   ```ts
   import type { Config } from 'tailwindcss'

   const config: Config = {
     content: [
       './app/**/*.{ts,tsx}',
       './components/**/*.{ts,tsx}',
       './lib/**/*.{ts,tsx}',
     ],
     theme: {
       extend: {
         colors: {
           bg: '#EFE8DC',
           gold: '#C8973A',
           dark: '#1a1a1a',
         },
         fontFamily: {
           switzer: ['var(--font-switzer)', 'sans-serif'],
           sans: ['var(--font-public-sans)', 'sans-serif'],
         },
       },
     },
     plugins: [],
   }

   export default config
   ```
   Key: `colors.bg` is `#EFE8DC`, `colors.gold` is `#C8973A`, `colors.dark` is `#1a1a1a`. Font families reference the CSS variables set by next/font.

4. **Update `app/globals.css`:**
   ```css
   @tailwind base;
   @tailwind components;
   @tailwind utilities;

   @layer base {
     body {
       @apply bg-bg text-dark font-sans antialiased;
     }
   }
   ```
   Remove all default Next.js boilerplate styles. The body gets beige background, dark text, Public Sans as default, antialiased rendering.

5. **Update `app/layout.tsx`:**
   ```tsx
   import type { Metadata } from 'next'
   import { switzer, publicSans } from './fonts'
   import './globals.css'

   export const metadata: Metadata = {
     title: 'Corazon - Fotografia & Videografia',
     description: 'Uma contadora de historias visual. Fotografia e videografia premium.',
   }

   export default function RootLayout({
     children,
   }: {
     children: React.ReactNode
   }) {
     return (
       <html lang="pt-BR" className={`${switzer.variable} ${publicSans.variable}`}>
         <body>{children}</body>
       </html>
     )
   }
   ```
   Key: `lang="pt-BR"`, both font CSS variables applied to `<html>`, body has no extra classes (globals.css handles defaults).

6. **Update `app/page.tsx`** to a minimal landing page that validates fonts and colors:
   ```tsx
   export default function Home() {
     return (
       <main className="min-h-screen flex flex-col items-center justify-center gap-8 px-6">
         <h1 className="font-switzer text-5xl font-bold tracking-tight text-dark">
           Corazon
         </h1>
         <p className="font-sans text-lg text-dark/70">
           Uma contadora de historias visual
         </p>
         <div className="flex gap-4">
           <span className="inline-block w-16 h-16 rounded-full bg-bg border-2 border-dark/10" />
           <span className="inline-block w-16 h-16 rounded-full bg-gold" />
           <span className="inline-block w-16 h-16 rounded-full bg-dark" />
         </div>
       </main>
     )
   }
   ```
   This page displays Switzer heading, Public Sans body text, and all three color tokens as visible circles for visual verification.

7. **Create `lib/utils.ts`** with the cn() utility:
   ```ts
   import { clsx, type ClassValue } from 'clsx'
   import { twMerge } from 'tailwind-merge'

   export function cn(...inputs: ClassValue[]) {
     return twMerge(clsx(inputs))
   }
   ```
